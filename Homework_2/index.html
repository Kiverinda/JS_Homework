1. Почему код дает именно такие результаты?

var a = 1, b = 1, c, d;
  1. c = ++a; alert(c); //2
  2. d = b++; alert(d); //1
  3. c = (2+ ++a); alert(c); //5
  4. d = (2+ b++); alert(d); //4
  5. alert(a); //3
  6. alert(b); //3

Ответ

  1. Переменная а увеличивается на 1 и присваивается переменной с
  2. Переменной d присваевается переменная b, после этого переменная b увеличивается на 1
  3. К двум прибавляется увеличенная на 1 переменная а, которая после первого пункта уже равна 2. Результат сложения присваивается переменной с
  4. К двум прибавляется переменная b, равная после второго пункта 2. Результат сложения присваивается переменной d После этого переменная b увеличивается на 1
  5. После 3 пункта переменная a равна 3
  6. После 3 пункта переменная b равна 3

### 7. *Сравнить null и 0. Попробуйте объяснить результат.

Решение

Сравним null и 0

  1. null > 0
  2. null == 0
  3. null >= 0

Нестрогое равенство и сравнения > <>= <= работают по-разному. Сравнения преобразуют null в число, рассматривая его как 0. Поэтому выражение (3) null>= 0 истинно, а null > 0 ложно. С другой стороны, для нестрогого равенства == значений undefined и null действует особое правило: эти значения ни к чему не приводятся, они равны друг другу и не равны ничему другому. Поэтому null == 0 ложно.
